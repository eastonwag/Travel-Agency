# Travel Agency Reservation System

A full-stack reservation system for a travel agency, built by integrating an existing **MySQL database** with a **Spring Boot REST API** back-end and an **Angular** front-end. The back-end exposes a RESTful API via Spring Data REST, allowing the Angular client to browse, create, and manage travel reservations with no custom controller boilerplate.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Language | Java 17 |
| Framework | Spring Boot 3.5 |
| Persistence | Spring Data JPA (Hibernate) |
| API Layer | Spring Data REST |
| Database | MySQL |
| Utilities | Lombok |
| Build Tool | Maven (Maven Wrapper included) |
| Front-End | Angular |

---

## Features

- **RESTful API** auto-generated by Spring Data REST — full CRUD endpoints exposed directly from JPA repositories
- **MySQL integration** — connects to an existing relational database schema for travel reservations
- **JPA / Hibernate ORM** — entity mapping and database interaction without manual SQL
- **Lombok** — reduces boilerplate with auto-generated getters, setters, constructors, and builders
- **Angular front-end** — consumes the REST API to provide a browser-based reservation interface
- **Maven Wrapper** — no local Maven installation required; use `./mvnw` to build and run

## Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/eastonwag/Travel-Agency.git
cd Travel-Agency
```

### 2. Configure the Database

Create a MySQL database for the application, then update the connection settings in `src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/your_database_name
spring.datasource.username=your_mysql_username
spring.datasource.password=your_mysql_password

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

### 3. Build and Run the Back-End

Using the Maven Wrapper (no local Maven install needed):

```bash
# macOS / Linux
./mvnw spring-boot:run

# Windows
mvnw.cmd spring-boot:run
```

The API will start at `http://localhost:8080`.

### 4. Run the Front-End

Navigate to the Angular project directory and install dependencies:

```bash
cd frontend   # adjust path as needed
npm install
ng serve
```

The Angular app will be available at `http://localhost:4200`.

## API Overview

Spring Data REST automatically exposes CRUD endpoints for each JPA repository. Once running, the full API is discoverable at the root endpoint:

```
GET http://localhost:8080/
```

Typical endpoints follow REST conventions, for example:

| Method | Endpoint | Description |
|---|---|---|
| GET | `/vacations` | List all vacation packages |
| GET | `/vacations/{id}` | Get a specific vacation package |
| POST | `/vacations` | Create a new vacation package |
| PUT | `/vacations/{id}` | Update a vacation package |
| DELETE | `/vacations/{id}` | Delete a vacation package |
| GET | `/customers` | List all customers |
| POST | `/checkouts` | Create a new reservation |

> Exact endpoints depend on the entity and repository names defined in the source code.
